## [未发布]

首个上线版本（v1）准备中。本节记录当前未发布候选的主要变更；是否已进入
`main`、是否具备正式发布条件，以 Git 历史、CI 与发布验收报告为准。

### 新增

- 中英文双语官网页面，覆盖联盟介绍、成员信息、工作组、资讯、参与路径与法律说明。
- 网络安全工作组详情页，以及参与说明、成员名单等配套页面。
- 网络安全人员开放计划的阶段性资讯页；申请入口仅在公开表单满足安全校验时显示。
- 项目协作、内容维护、部署运维、设计系统与发布验收文档。
- GitHub Pages 与容器两条部署路径，以及对应的 CI 质量门禁。
- 经当前公开资料核对的成员信息；未获核验的组织关系、资质与背书保持中性或不展示。

### 变更

- **统一正式 canonical 为 `https://www.zgc-llm.org.cn`（带连字符）**。计划由裸域、
  `zgc-llm.cn`、`zgc-llm.net` 以及无连字符的防御性域名
  `zgcllm.org.cn`、`zgcllm.cn`、`zgcllm.net` 单跳 301 至正式主域；实际启用须在
  DNS、TLS 与 GitHub Pages 配置完成后复验。
- 重构全站信息层级、响应式导航、CTA 与状态反馈，统一中英文术语和专业表达。
- 参与入口采用三目标完整性校验；相关 CTA 仅在三个公开链接均通过仓库配置校验时显示。
  发布前还须逐项复验外部表单可访问且允许匿名提交，未满足时保持关闭。官网不接收或保存
  申请人信息。
- 主题跟随操作系统偏好，不提供会造成状态误解的站内主题开关。
- 固定 Node.js、pnpm 与关键 CI Action 版本，统一冻结锁文件、缓存、并发取消、最小权限
  与部署职责边界。
- 建立单元、集成、跨浏览器 E2E、可访问性、SEO、链接、静态服务器与 Lighthouse 回归矩阵。

### 修复

- 修正 `CNAME`、运行时配置、工作流、容器与文档中的域名漂移，避免错误 canonical。
- 移除未经核验的上线承诺、具名合作关系、资质背书、英文全称与联系动作。
- 补齐键盘焦点、移动菜单焦点约束、跳转链接、错误页、hreflang 与静态服务安全边界。
- 治理文档、测试与格式化债务，并把相应检查纳入 CI。
