## 两轴模型

- **申请主体**（单选）：专家个人 / 机构
- **意向**（多选）：加入联盟 / 参与网络安全工作组

一条记录一行，靠这两个字段区分，取代旧表「X / X（工作组申请）」重复字段。

## 表字段（24）

**A 区 · 问卷写入**：申请编号(自动编号)、提交时间(创建时间)、申请主体(单选·必填)、意向(多选·必填)、姓名/联系人(文本·必填)、机构名称(文本)、机构类型(单选 企业/高校/科研院所/实验室/其他)、职务/身份(文本)、邮箱(文本·email·必填)、手机号(文本·phone·选填)、所在城市(文本)、专业/合作方向(多选)、能力标签(多选 模型/芯片/算力/数据/平台/行业应用/安全)、投入意愿/合作说明(文本)、代表性成果/材料(附件)、推荐人/来源(文本)、数据授权确认(勾选·必填)

**B 区 · 运营流转（内部，不进表单）**：状态(单选 待审核/初审中/复审中/已通过/已驳回/待补充材料)、评审人(人员)、评审意见(文本)、跟进记录(文本)、结果通知时间(日期)、目标工作组(单选 网络安全工作组)、会员级别(文本，通过后定级)

> 注：原设计的「来源入口」已去除——单一表单只有一个分享链接，无法自动区分官网机构/网安组入口。

## 表单分支流程（飞书问卷）

- **第1页**：申请主体(单选·必填)、意向(多选·必填)
- **第2页·专家**（申请主体=专家个人显示）：姓名、邮箱、手机号、所在城市、职务/身份、所在机构、专业方向(多选)、能力标签(多选)、投入意愿、成果材料、推荐来源
- **第2页·机构**（=机构显示）：机构名称、机构类型、联系人姓名、联系人职务、邮箱、手机号、合作方向(多选)、能力标签、合作说明、资质材料、推荐来源
- **第3页**：补充说明、数据用途授权(必勾)

## 待飞书后台 UI 收尾

1. **补 3 个问题**：邮箱(必填)、手机号(选填)、数据授权确认(必填勾选)——CLI 加 email/phone/checkbox 不稳，UI 里加。
2. **条件显示**：按「申请主体」设专家/机构分支（API 不支持，UI 专属）。
3. **必填补齐**：机构名称/机构类型（机构分支必填）。
4. **排序/分页**：按上面三页顺序。
5. **挂《数据用途说明》**：文案由运营出，问卷内以链接或段落呈现；设为外部/匿名可填的公开分享链接。
6. 取得分享链接后，配置到官网 `NEXT_PUBLIC_APPLICATION_URL`（GitHub 仓库 Variable）。
