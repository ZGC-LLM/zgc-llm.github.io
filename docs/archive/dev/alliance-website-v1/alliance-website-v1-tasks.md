# 开发任务: 中关村自主大模型产业联盟官网一期

> **更新补记（2026-07-16）**：本文件为 alliance-website-v1 开发期的历史设计记录。项目已在 PR #4 中移除 Payload CMS 与 PostgreSQL，收敛为纯静态官网——公开内容由 `src/content/` 驱动，加入申请通过飞书表单外链承接。以下涉及 Payload / 数据库的内容仅作历史留存，不代表当前架构。

## 1. 执行规则

- Planning 已统一确认；用户随后开启隔夜执行，Execution 按 `run-owl` 自动完成。
- 每个任务遵循 Red → Green → Refactor；先补充或更新失败测试，再实现最小功能。
- 所有实现均在 `codex/alliance-website-v1` worktree 中进行。
- 每组完成后运行该组测试及 `pnpm typecheck`、`pnpm lint`，通过自动里程碑评估后进入下一组。
- 不在本任务中提交真实飞书数据、生产密钥、未授权 Logo 或虚构成员信息。
- 网络安全生态保持厂商中立，评审时检查并阻止任何单一基础模型品牌绑定表述。

### Group 1：内容基础、站点外壳与 SEO

#### T-002 建立类型化内容与申请目标配置

**依赖：** T-001  
**并行：** 否；先冻结内容与配置接口，供 T-003、T-004 使用  
**目标：** 提供页面统一使用的内容模型、导航、联系信息和飞书外链配置。

**测试先行：**

- 扩展 `tests/unit/site-config.test.ts`，覆盖导航唯一性、公开路由和双申请目标。
- 新增申请 URL 校验测试，覆盖 HTTPS、缺失、非法协议和畸形 URL。
- 新增新闻数据测试，覆盖唯一 slug、日期、发布过滤和厂商中立表述。

**实施：**

- 新增 `src/types/content.ts` 和 `src/content/*`。
- 重构 `src/config/site.ts`，集中管理站点信息、导航与申请目标。
- 使用保守的占位内容，不添加未经确认的成员、数据或成果。

**验收：**

- TypeScript 严格模式通过，无 `any`。
- 同类 CTA 只读取一个配置来源。
- 非 HTTPS 或缺失链接不会被输出为可点击申请入口。
- 内容与测试中不将网络安全生态绑定到单一基础模型品牌。

**实施：**

- 新增 `src/components/site/*` 共享组件。
- 更新前台根布局使用共享外壳；Payload 路由组保持不变。

**验收：**

- 桌面与移动导航均可通过键盘使用。
- 组件中无散落硬编码品牌色，交互状态完整。
- 主 CTA 为机构生态共建，次 CTA 为专业用户加入。

**验收：**

**Group 1 闸门：** 运行相关单元/组件测试、`pnpm typecheck`、`pnpm lint`；完成 standard 两阶段评审后等待用户确认。
